# AssetControl

**AssetControl** - это дипломный проект на тему:

**"Разработка автоматизированной системы учёта и контроля оборудования предприятия"**

Если объяснять совсем просто, это сайт для предприятия, где можно видеть:

- какое оборудование есть в компании;
- где оно находится;
- кому оно выдано;
- что сейчас в ремонте;
- что просрочено;
- что потеряно или списано;
- кто и когда что сделал в системе.

Простая аналогия: раньше всё оборудование могли вести в Excel или в бумажном журнале. Это неудобно, потому что файл можно забыть обновить, бумагу можно потерять, а устную передачу ноутбука потом сложно доказать. AssetControl заменяет такой хаос одной понятной системой.

## Главная идея проекта

Проект показывает не просто таблицу с оборудованием, а полный путь оборудования на предприятии.

Например:

1. Ноутбук появился в базе.
2. Его положили на склад.
3. Его выдали сотруднику.
4. Потом сотрудник вернул ноутбук.
5. Если ноутбук сломался, его отправили в ремонт.
6. Во время инвентаризации проверили, где он находится.
7. Если оборудование устарело, его можно списать.
8. Все важные действия сохраняются в журнале.

Это похоже на карточку пациента в больнице: важно знать не только имя пациента, но и всю историю: когда поступил, кто лечил, что назначали и какой результат. Здесь такая же логика, только для оборудования.

## Что умеет система

- Вход в систему по логину и паролю.
- Разные роли пользователей: администратор, менеджер, сотрудник, аудитор.
- Реестр оборудования: ноутбуки, мониторы, принтеры, серверы и другое.
- Карточка оборудования с владельцем, локацией, стоимостью, гарантией и историей.
- Выдача оборудования сотруднику.
- Возврат оборудования.
- Ремонтные заявки.
- Инвентаризация.
- Dashboard с главными цифрами.
- Журнал действий, где видно кто и что сделал.
- Уведомления.
- CSV-отчёты для Excel.
- PDF-отчёты для печати и показа руководителю.
- Мониторинг через Grafana.
- Тесты backend и frontend.
- Запуск всего проекта через Docker.

## Роли пользователей

| Роль | Простое объяснение |
|---|---|
| ADMIN | Главный пользователь. Может всё. |
| MANAGER | Управляет оборудованием, выдачами, ремонтами и инвентаризацией. |
| INVENTORY_MANAGER | Отвечает за учёт и инвентаризацию. |
| EMPLOYEE | Обычный сотрудник. Может смотреть только доступную ему информацию. |
| AUDITOR | Проверяющий. Смотрит отчёты и журнал действий. |
| VIEWER | Наблюдатель. Может смотреть обычные данные и отчёты без изменения, но не видит журнал действий. |

Простая аналогия: как в университете. Есть администратор системы, преподаватель, студент и проверяющий. Все заходят в одну систему, но права у всех разные.

## Из чего состоит проект

Проект состоит из нескольких частей.

| Часть | Что это значит простыми словами |
|---|---|
| Frontend | То, что видит пользователь в браузере. Кнопки, таблицы, страницы. |
| Backend | Серверная часть. Она принимает запросы, проверяет права и работает с базой. |
| PostgreSQL | База данных. Там хранятся пользователи, оборудование, выдачи, ремонты и отчёты. |
| Prisma | Удобный слой для работы backend с базой данных. |
| Docker Compose | Команда, которая запускает весь проект сразу. |
| Prometheus и Grafana | Мониторинг. Показывает, работает ли система и какие в ней важные показатели. |

Если сравнить с рестораном:

- frontend - это зал и меню для клиента;
- backend - это кухня;
- база данных - это склад продуктов и журнал заказов;
- Docker - это способ быстро открыть весь ресторан;
- Grafana - это табло для управляющего.

## Быстрый запуск

Откройте терминал в папке проекта и выполните:

```bash
docker compose up --build
```

После запуска открыть:

| Что открыть | Адрес |
|---|---|
| Сама система | http://localhost:6347 |
| Проверка backend | http://localhost:5847/api/health |
| Grafana | http://localhost:5240 |
| Prometheus | http://localhost:4747 |
| pgAdmin | http://localhost:8571 |

## CSV и PDF отчёты

В системе есть два вида отчётов.

### CSV

CSV нужен для Excel.

Например, бухгалтер или менеджер может открыть CSV в Excel и сделать фильтрацию, сортировку или дополнительный расчёт.

### PDF

PDF нужен для официального вида.

Его можно:

- распечатать;
- отправить руководителю;
- приложить к отчётности;
- показать комиссии на защите.

PDF-отчёты выглядят как аккуратные корпоративные документы: есть заголовок, дата, пользователь, summary-блоки, таблицы, номера страниц и footer.

Доступные отчёты:

| Отчёт | CSV | PDF |
|---|---|---|
| Оборудование | есть | есть |
| Выдачи | есть | есть |
| Ремонты | есть | есть |
| Инвентаризация | есть | есть |
| Audit log | нет | есть, только ADMIN/AUDITOR |

Простая фраза для защиты:

> CSV используется для обработки данных в Excel, а PDF - для официального представления и печати отчётов.

## Мониторинг Grafana

Grafana в этом проекте нужна не для сложного промышленного DevOps-мониторинга. Здесь она показывает понятные вещи для диплома:

- работает ли база данных;
- сколько запросов получает API;
- есть ли ошибки;
- сколько оборудования в системе;
- сколько оборудования доступно;
- сколько выдано;
- сколько в ремонте;
- сколько просроченных выдач;
- сколько записей в audit log.

Простая аналогия: Grafana - это панель приборов в автомобиле. Водитель видит скорость, топливо и ошибки. Здесь администратор видит состояние системы и основные бизнес-показатели.

## Честные ограничения

Это дипломный MVP, а не готовая промышленная система для реального завода на тысячи сотрудников.

Что можно добавить в будущем:

- QR-коды на оборудование.
- Отдельные PDF-акты выдачи и возврата.
- Загрузку файлов и фото.
- Более сложные уведомления в реальном времени.
- Более глубокие e2e-тесты.
- Более сложную систему refresh-токенов.

Как это объяснить комиссии:

> В дипломе я сфокусировался на главной задаче: показать учёт и контроль оборудования. Дополнительные функции, такие как QR-коды и файловые вложения, логично добавить на следующем этапе развития системы.

## Короткое описание проекта для комиссии

Можно сказать так:

> Мой проект называется AssetControl. Это система для учёта и контроля оборудования предприятия. Она помогает видеть, какое оборудование есть в компании, где оно находится, кому оно выдано, что находится в ремонте и какие действия выполняли пользователи. Система заменяет ручной учёт в Excel и бумажных журналах на единый цифровой инструмент.
